# WO-000 - Bootstrap Mora and make the toolchain real

Feature: F-000 in FEATURES.md - Branch: `feat/000-toolchain` - Diff cap: 400 lines (lockfile excluded)

## Objective

Turn the protected scaffold into a real, testable Next.js product repo and encode the Owner-approved product definition. This is a bootstrap PR: F-000 cannot pass CI while the scaffolded scripts are placeholders, so the approved Define artifacts ride with the toolchain instead of pretending a docs-only PR could merge.

## Acceptance criteria

1. `PRODUCT.md` defines Mora for adult intermediate language learners: record up to 30 minutes of natural target-language speech, analyze native audio without an STT/transcription pipeline, prioritize useful grammar and vocabulary feedback, release at most 20 new corrections per day, and teach through contextual recall and short lessons. Non-goals: live AI conversation, full transcripts, pronunciation scoring, accounts/social features, and cloud sync in v1.
2. `DECISIONS.md` appends the approved calls: Next.js 16 + strict TypeScript; browser WebRTC to `gpt-realtime-2.1`; no transcription endpoint or stored transcript; local-first IndexedDB; API key server-only through ephemeral session credentials; at most 20 new cards/day with a 10 grammar/10 vocabulary target that may backfill from the other category; contextual productive-recall cards scheduled by `ts-fsrs`; recordings are not stored server-side and the local retry copy is deleted after successful analysis.
3. `FEATURES.md` marks F-000 `built` and adds ordered backlog items with testable acceptance criteria: F-001 shell/settings, F-002 resilient 30-minute recorder, F-003 native-audio analysis, F-004 findings inbox/daily release, F-005 FSRS review, F-006 mini-lessons, F-007 privacy/QA/deploy.
4. Create lowercase `learnings.md` with dated L-001... entries for: manual Define substitution because F-008 is absent; first-PR coupling between approved artifacts and F-000; review status still shares the factory credential; scaffolded products have no mfactory source-version or sync path. State evidence and a candidate factory improvement without claiming the improvement is approved.
5. Replace placeholder scripts with real `lint`, `typecheck`, and `test` commands. Use Next.js 16.2.10, React, strict TypeScript, ESLint, Vitest, Testing Library, jsdom, and V8 coverage. `npm run test` emits `coverage/coverage-summary.json`; commit `.mfactory/coverage-baseline.json` and ignore generated coverage/build output.
6. Add only a minimal accessible page identifying Mora and one meaningful render test. Product UI and design work are F-001, not this feature.
7. `npm run lint`, `npm run typecheck`, `npm run test`, `node .mfactory/ci/coverage-ratchet.mjs`, and `git diff --check` pass. Keep the non-lockfile diff at or below 400 lines.
